# Known Cross-Tool Conflicts

When multiple guardrail tools run on the same project, some interactions produce unexpected behavior. This catalog documents known conflicts and their resolutions.

---

## Prettier × Column-Aligned Tables

**Symptom:** Prettier strips trailing whitespace from markdown tables. If a tool or script generates column-aligned tables with load-bearing trailing whitespace, Prettier reformats them on every save, creating an infinite loop between the generator and the formatter.

**Resolution:** Add affected files to `.prettierignore` with a comment naming the generator:

```
# Column-aligned tables generated by docs:sync
docs/auto-generated.md
```

---

## Prettier × MDX

**Symptom:** Prettier's MDX formatting can produce parse errors with certain JSX patterns, especially nested components with complex props.

**Resolution:** Pin the prettier MDX plugin version and test before upgrading. If issues persist, add problematic MDX files to `.prettierignore`.

---

## Knip × Dynamic Registries

**Symptom:** Knip reports packages or exports as unused when they're loaded dynamically via string-keyed lookups (e.g., plugin registries, route maps, service locators).

**Example:**
```typescript
// Knip can't trace this — it's a string lookup
const handler = handlers[request.type]; // 'handlers' values are never statically imported
```

**Resolution:** Add to `knip.json` ignore list with a comment naming the registry file:

```json
{
  "ignoreDependencies": ["dynamic-plugin-a", "dynamic-plugin-b"]
}
```

Or use the `entry` field to include the registry file as an entry point.

---

## Boundaries × Test-File Conventions

**Symptom:** Boundary violations in test files, or test files not being excluded from boundary checks.

**Root cause:** Different projects use different test-file conventions (`__tests__/**`, `*.test.ts`, `*.spec.ts`, `tests/**`). The default boundaries config only excludes `__tests__/**`.

**Resolution:** Ensure your boundaries config excludes all your test-file patterns:

```javascript
{
  files: ['packages/*/src/**/*.ts'],
  ignores: [
    '**/__tests__/**',
    '**/*.test.ts',
    '**/*.spec.ts',
    '**/tests/**',
    '**/__mocks__/**',
  ],
  // ... boundaries rules
}
```

## TypeScript Path Aliases × ESLint Resolver

**Symptom:** ESLint reports false clean results — rules like `import-x/default` and `import-x/named` don't fire, even when imports are genuinely wrong. Or, ESLint reports false errors on valid path-aliased imports.

**Root cause:** Both TypeScript (`tsconfig.json` paths) and ESLint (`eslint-import-resolver-typescript`) must be configured. If the ESLint resolver isn't set up, `import-x` rules silently skip resolution and produce no output.

**Resolution:** Install and configure the resolver:

```bash
npm install -D eslint-import-resolver-typescript
```

Add to `eslint.config.js`:

```javascript
{
  settings: {
    'import-x/resolver': {
      typescript: {
        project: './tsconfig.json',
      },
    },
  },
}
```

## docs-check × Narrative Prose

**Symptom:** The `docs-check` script flags hypothetical paths used in explanatory text (e.g., "if your project has `src/infrastructure/validators/`...").

**Resolution:** Add a `<!-- docs-check-ignore -->` comment on the line before the hypothetical path:

```markdown
<!-- docs-check-ignore -->
For example, if your project has `src/infrastructure/validators/boot.ts`, wire it as...
```

---

## ESLint Config Prettier × ESLint Rules

**Symptom:** ESLint auto-fix changes formatting that Prettier then changes back on save, creating an infinite loop.

**Root cause:** `eslint-config-prettier` must be the LAST item in the ESLint config to properly disable formatting rules.

**Resolution:** Ensure this ordering:

```javascript
export default tseslint.config(
  ...otherConfigs,
  eslintConfigPrettier,  // ← MUST be last
);
```
